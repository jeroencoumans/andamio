Views

Learn how to setup your views

By default, Andamio comes with support of three default views. Each view has its own header and scrollable content area.

Parent view
This is the default view you see when opening Andamio. It's main purpose is to show the user the primary actions, e.g. a list of articles.
Child view
This view will slide in from the right. It's meant to show a full view of the whatever item the user clicks on in the overview list, e.g. a full article.
Modal view
This view will slide up from the bottom. It's main purpose is to present the user a set of secondary actions or content, e.g. preferences or help text.

The content of a view is setup as follows:



    
Be sure to include all the classes as shown to allow Andamio to do its magic. 

    

        
.page-header

        
This contains the view's header bar at the top. By default it's fixed so it won't scroll. 

        
.page-logo

        
Here you can put in the title of the view, or a logo. It's configured at a maximum width of 140px (configurable in your main.less).

        
.js-title

        
This class allows actions to set the title of the view they're opening. Feel free to leave it out if you're using a fixed logo that should be placed on each view.

        
.button-header-wrapper

        
This class will create a wrapper for a button on the left side of the page header. Make sure to also include the .button-header element! The wrapper is used for positioning and the action, while the inner element is used for styling. It's setup this way to allow a bigger click target.

        
.button-header-wrapper-alternate

        
Same as .button-header-wrapper, except for creating a button on the right side of the page header. 

        
.page-content

        
This holds the scrollable content and is positioned under the header. It will fill up the remaining space.

        
.js-content

        
This is how actions know where to put the content; by default it goes to the page-content, but in some cases you may put this somewhere else. 

        
.overthrow

        
This makes sure the content is scrollable.
