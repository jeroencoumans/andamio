How to setup a new page

Andamio can run perfectly fine with only 1 main index.html. Start from this template (found in boilerplate/index.html:



    
By default, the <html> element should always start with the .website class. Andamio will check the environment it runs in and will adjust this class for the following cases:

    
    
        
  • .website: when run as mobile website, based on user agent and screen size
    • 
        
  • .webapp: for mobile web application, when run with the ?webapp=1 parameter, or when running as standalone application.
    • 
        
  • .desktop: for desktop website
    • 
    


    
The .viewport class is mandatory for the <body> element.

    
Nothing needs to be adjusted in the head except the <title>!

1. Page views

    
The page-view is a wrapper for the parent-view and any (optional) child-view. 

    
By default, Andamio supports a two-level hierarchy. The default displayed page is called the parent-view, while any sub-page is called the child-view. The default HTML structure looks like this (minus any content):





    
The animation is controlled with the following classes: .active-view and .view-hidden. 


    
Actions:

    

        
.action-push

        
Opens a page in the child-view. You can also open child pages from a child page, but no animation will be set and going back (using .action-pop) will still go back to the same parent page.


        
.action-pop

        
Opens a page in the parent-view. If we're on a child view, it will first transition back. 

    

    
By default, Andamio will use the text of the link as the title, and the href of the link as the page to load. You can override these by using a data-url and data-title attribute.









    
2. Page navigation

    
Contains navigation elements. On mobile, it will appear as a slide-out menu from the left. On desktop, it will remain fixed to the left.





    
The page-navigation can be set to allow scrolling by adding the .overthrow class (included by default). 

    
Andamio will set the .active class on the clicked navigation item. 


    
Actions:

    

        
.action-nav-item

        
Load the linked page into the parent view.


        
.action-show-nav

        
Shows the navigation. Used in the menu link in the header.


        
.action-hide-nav

        
Hides the navigation. The .page-navigation-toggle makes use of this by overlapping the content when the menu is open. 

    









    
4. Page loader

    
When any content takes longer than half a second to load, a spinner will appear on top of the content. The loader contains the spinner. 













    
3. Modal views

    
Wrapper for any modal views. These will slide up from the bottom. Note that you can open as many modals as you want, they will just replace each other's content (so no modal stack is yet available).





    
Actions:

    

        
.action-show-modal

        
Shows the modal. You can set default content inside the modal, or you can specify a link to load a page inside the modal. 

        

        
.action-hide-modal

        
Hides the modal window.

    









    
5. Page alerts

    
This will display alert messages inside an overlay on top of the page.
