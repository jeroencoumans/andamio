Page structure

How to setup a new page

Andamio can run perfectly fine with only 1 main index.html. Start from this template (found in boilerplate/index.html:

<!DOCTYPE html>
<html lang="nl" class="website">
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">

    <!-- Set these for mobile-optimized website that doesn't require the user to zoom in. Required for webapp -->
    <meta name="viewport" content="initial-scale=1.0, maximum-scale=1.0, user-scalable=0">

    <!-- Set these to allow the mobile website/webapp to be installed on the home screen -->
    <meta name="apple-mobile-web-app-capable" content="yes">
    <meta name="apple-mobile-web-app-status-bar-style" content="black">

    <!-- Set a shorter title for iOS6 devices when saved to home screen -->
    <meta name="apple-mobile-web-app-title" content="Andamio">
    <title>Andamio</title>

    <!-- Include the main bundle or create your own -->
    <link rel="stylesheet" type="text/css" href="style/main.css">
</head>

<body class="viewport">

</body>
</html>

The <html> element can contain one of the following classes:

  • .website: for mobile website
  • .webapp: for mobile web application
  • .desktop: for desktop website
  • .desktop.webapp: for desktop web application

The .viewport class is mandatory for the <body> element.

The <body> can contain one of the following elements:

1. Page views

The page-view is a wrapper for the parent-view and any (optional) child-view. 


    <div id="page-view" class="page-view">
        <div id="parent-view" class="parent-view active-view">

        </div>

        <div id="child-view" class="child-view view-hidden">

        </div>
    </div>

2. Page navigation

Contains navigation elements. On mobile, it will appear as a slide-out menu from the left. On desktop, it will remain fixed to the left.

The .action-hide-nav link overlaps the content when the navigation is open. This makes it easier to close the menu just by tapping anywhere outside it.


<div id="page-navigation" class="page-navigation overthrow">
    <h2 class="navigation-header">Docs</h2>
    <a class="navigation-item action-nav-item active" href="doc/getting-started.html">Getting started</a>
</div>

<a href="javascript:void(0)" class="page-navigation-toggle action-hide-nav"></a>

3. Modal views

Wrapper for any modal views. These will slide up from the bottom.


<div id="modal-view" class="modal-view view-hidden">

</div>

4. Page loader

When any content takes longer than half a second to load, a spinner will appear on top of the content. The loader contains the spinner. 


<div id="loader" class="loader" style="display:none">
    <img src="img/spinner-dark.gif" class="spinner" width="32" height="32">
    <div class="loader-text"></div>
</div>

5. Page alerts

This will display alert messages inside an overlay on top of the page.


<div id="page-alert" class="page-alert action-hide-alert" style="display:none">
    <a href="#" class="action-refresh button">Refresh</a>
</div>
